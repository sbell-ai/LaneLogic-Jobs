**Objective**  

Add a **Social Publishing** admin module that feels native in the existing admin dashboard. It must support creating, editing, scheduling, and queueing social posts for **Jobs**, **Blog posts**, and **Resources** using **Zapier-first** (Option A). **No direct social network APIs** in MVP.

**MVP platforms:** `linkedin`, `facebook_page`, `instagram_business` (schema must be extensible for `x`, `threads`, `tiktok`).

**Confirmed UX choice:** Share modal defaults to **Post now**, with a **Schedule** toggle that reveals datetime picker.

**Critical rule (Published-only):** Only entities with `isPublished === true` can be shared/queued.

Repo specifics (must implement in these files):

- Express entry: `server/index.ts`
- Routes: `server/routes.ts`
- Drizzle config: `drizzle.config.ts`
- All tables: `shared/schema.ts` (single schema file)
- Client routing: Wouter, routes in `client/src/App.tsx`
- Admin section switch: `client/src/pages/dashboard/AdminDashboard.tsx`
- Admin sidebar links: `client/src/pages/dashboard/DashboardLayout.tsx` (`adminLinks` array)

## 0) Required fixes / clarifications (must adjust plan)

### F1) ID type consistency

Do **not** invent an odd ID type. Match existing conventions in `shared/schema.ts`.

- If the project already uses UUID column types, use UUID for `social_posts.id`.
- If the project uses string IDs with uuid defaults, that’s acceptable, but then treat IDs as **strings everywhere** (storage, routes, client).
- `providerRequestId` must be a uuid string as well.

### F2) “Share to Social” must be on the dedicated *edit pages* (MVP)

You confirmed jobs/blog/resources each have dedicated admin edit pages.

- Implement the Share button and modal on those edit pages.
- Do not rely on adding share buttons only inside list tabs (`AllJobsTab`, etc.). (Share-from-list can be a later enhancement.)

### F3) Queue-time copy policy must be explicit

Policy (recommended):

- At queue time, for each selected platform:
    - If `copyByPlatform[platform]` is missing/empty and `copyMaster` exists, auto-derive from `copyMaster`.
    - If both are missing, block with a specific error.

Server must enforce this in `/queue` (even if client validates).

### F4) Cancel semantics (Zapier-first constraint)

MVP: allow cancel **only** for `status="draft"`.

- If `status="queued"`, return a clear message that queued posts cannot be canceled in MVP.

### F5) Secure the Zapier callback endpoint

`providerRequestId` matching alone is not sufficient.

- Add env var `ZAPIER_CALLBACK_SECRET`
- Require header `X-Zapier-Secret` on callback
- Reject missing/invalid secret with 401

### F6) Canonical public URL generation must be centralized

Create one server helper: `getPublicEntityUrl(entityType, entity)` and use it everywhere so URLs never drift.

## 4) Server endpoints (Express)

**File:** `server/routes.ts`

### Environment variables

- `ZAPIER_SOCIAL_POST_WEBHOOK_URL`
- `ZAPIER_CALLBACK_SECRET`

### Auth requirement (admin)

All `/api/admin/*` endpoints must require admin auth (match existing pattern).

### Published-only enforcement (CRITICAL)

For Jobs/Blog/Resources, “Published” means:

- `entity.isPublished === true`

Enforce this in BOTH:

- `POST /api/admin/social-posts`
- `POST /api/admin/social-posts/:id/queue`

If not published, return **409** with:

```json
{
  "message": "Only published items can be shared to social.",
  "errors": [{ "field": "isPublished", "reason": "not_published" }]
}
```

#### Endpoints

**A) Create draft**

`POST /api/admin/social-posts`

- Load entity by entityType/entityId
- If `isPublished !== true`, return 409 (above)
- Compute:
    - `entityUrl` via `getPublicEntityUrl(...)`
    - `linkUrl` via `buildLinkUrl(entityUrl)`
    - `titleSnapshot`, `imageUrl`
- Create `copyByPlatform` via `generateDefaultCopy`
- Insert SocialPost with `status="draft"`, `createdBy=req.user.id`
- Return created record

**B) Update draft**

`PATCH /api/admin/social-posts/:id`

- Allow updates: `platforms`, `scheduledAt`, `copyMaster`, `copyByPlatform`, `imageUrl`
- Validate:
    - `platforms.length >= 1`
    - `validatePlatformCopy(...)` returns no errors
- Return updated record or 400/409 with structured errors

**C) List**

`GET /api/admin/social-posts?status=&entityType=&platform=`

- Return list for Social Queue

**D) Queue to Zapier**

`POST /api/admin/social-posts/:id/queue`

- Allowed status: `draft` or `failed`
- Re-load the underlying entity and enforce `isPublished === true` (return 409 if not)
- Validate:
    - platforms present
    - copy for each platform exists OR can be derived from copyMaster (per F3)
    - linkUrl exists
    - char limits pass
- Generate `providerRequestId` (uuid string)
- POST payload to `ZAPIER_SOCIAL_POST_WEBHOOK_URL`:

```json
{
  "socialPostId": "...",
  "providerRequestId": "...",
  "scheduledAt": "... or null",
  "platforms": [...],
  "copyByPlatform": {...},
  "linkUrl": "...",
  "imageUrl": "...",
  "entity": {"type":"job|blog|resource","id":"...","title":"..."}
}
```

- On success:
    - set `status="queued"`, store `providerRequestId`, store response metadata in `providerResponse`
- On failure:
    - set `status="failed"`, set `lastError`, return a specific error message (never generic)

**E) Retry (alias of queue)**

`POST /api/admin/social-posts/:id/retry`

- Same logic as queue, only if status is failed/draft

**F) Cancel (MVP restriction)**

`POST /api/admin/social-posts/:id/cancel`

- Only allow cancel when `status="draft"`.
- Otherwise return a clear error: queued posts cannot be canceled in MVP.

**G) Zapier callback (secured)**

`POST /api/integrations/zapier/social-posts/callback`

- Require header `X-Zapier-Secret` matching env `ZAPIER_CALLBACK_SECRET`, else 401
- Verify `providerRequestId` matches the stored one, else 409
- Update `status` to `sent` or `failed`, store `providerJobId`, `lastError`, and callback payload in `providerResponse`

**H) Connections support**

- `POST /api/admin/social-posts/test-webhook`
    - Send a test payload to `ZAPIER_SOCIAL_POST_WEBHOOK_URL` and return status + response
- `GET /api/admin/social-posts/webhook-status`
    - Return `{ configured: boolean }` without exposing secret values
