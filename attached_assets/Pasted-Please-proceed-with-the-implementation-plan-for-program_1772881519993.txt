Please proceed with the implementation plan for programmatic SEO job pages, but apply the following adjustments to ensure filtering accuracy and safe routing.

GENERAL GOAL
Implement programmatic SEO pages using the URL format:

/[category]-jobs-[state]

Examples:

/cdl-jobs-texas

/tanker-jobs-ohio

/flatbed-jobs-florida

The pages should filter jobs using keyword matching and fall back to Driver jobs if fewer than 3 matches exist.

ADJUSTMENTS TO THE CURRENT PLAN

Improve keyword matching logic (T002)

Do NOT match against the full job description first because it can cause inaccurate results.

Filtering priority should be:

Primary fields:

job.title

job.category

Secondary field:

job.description (only used if primary matches are fewer than 3)

Implementation logic:

Build a lowercase string using job.title + job.category.

Check if any keyword from category.match[] appears in that string.

If fewer than 3 results are found, allow matches from job.description as well.

If still fewer than 3 results, fall back to Driver jobs in that state.

Example fallback keyword:
"driver"

Add slug validation before rendering SEO pages (T003)

The router will catch all / paths, so validate before rendering.

Rules:

Only process the page if the slug contains "-jobs-"

Split the slug on "-jobs-" to get categorySlug and stateSlug

Confirm categorySlug exists in JOB_CATEGORIES

Confirm stateSlug exists in US_STATES

If any of these fail, fall through to the CMS or NotFound handler.

Example logic:

if (!slug.includes("-jobs-")) return null;
